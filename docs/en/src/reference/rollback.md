# `rollback` Command

The `rollback` subcommand restores previous configurations from the versioned backups generated by `apply`. It allows reverting changes in a granular or massive manner.

## Syntax

```bash
vallumix rollback [FLAGS] [--control-id <id> | --session <session>]
```

## rollback-Specific Flags

| Flag | Description |
|---|---|
| `--control-id <id>` | Reverts only the specified control (for example, `5.2.4`). |
| `--session <session>` | Reverts all controls from a session. Values: `last` or timestamp (`2026-05-01T09-15-33`). |
| `--dry-run` | Shows what would be restored without doing it. |
| `--verbose` | Shows detail of each restored file. |
| `--quiet` | Silences console output. |
| `--report` | Rollback report formats. |
| `--output` | Path prefix for reports. |

```warning
You must specify exactly one of `--control-id` or `--session`. If you specify neither, or if you specify both, Vallumix returns a configuration error (code `2`).
```

## Behavior

1. **Validation:** root, backup directory existence.
2. **Session resolution:** if `--session last` is used, selects the directory with the most recent timestamp.
3. **Integrity verification:** recalculates SHA-256 of each backup file and compares it with `checksums.sha256`.
4. **Restoration:** copies backup files to their original locations.
5. **Post-check:** executes `check` on restored controls to confirm the system returned to the previous state.
6. **Report:** documents which controls were restored, which failed integrity verification, and the final state.

## Examples

### Rollback the Last Problematic Control

```bash
sudo vallumix rollback --control-id 5.2.4
```

Restores only control `5.2.4` from the most recent backup session.

### Rollback the Latest Complete Session

```bash
sudo vallumix rollback --session last
```

Restores all controls applied in the most recent session.

### Rollback a Specific Session

```bash
sudo vallumix rollback --session 2026-05-01T09-15-33
```

Restores all controls from the session with the exact timestamp.

### Dry-Run Rollback

```bash
sudo vallumix rollback --session last --dry-run --verbose
```

Shows which files would be restored, from which backups, and the expected post-check result, without modifying anything.

## Control States in rollback

| State | Meaning |
|---|---|
| `Restored` | The control was successfully reverted; post-check confirms the previous state. |
| `RestoreFailed` | Restoration failed: corrupt backup file, insufficient permissions, or post-check mismatch. |
| `IntegrityCheckFailed` | The backup SHA-256 does not match the recorded one; restoration is aborted for this control. |
| `NotFound` | No backup exists for the requested control in the indicated session. |

```danger
If a control shows `IntegrityCheckFailed`, do not attempt to manually restore the backup file without first investigating why the hash changed. It could indicate disk corruption, file tampering, or a bug in the backup process. Contact the system administrator or security team.
```
